# Hanzo Studio RFCs

RFC (Request for Comments) process for proposing and discussing significant changes to Hanzo Studio.

## What is an RFC?

An RFC is a design document providing information to the Hanzo Studio community, describing a new feature or significant change to existing functionality.

## When to submit an RFC

Submit an RFC when you want to propose:

- New features or APIs
- Significant changes to existing features
- Breaking changes
- Architecture or design changes
- Process or governance changes

## RFC Process

1. **Draft** - Create a new file in `rfcs/` using the template
2. **Submit** - Create a pull request
3. **Discussion** - Community provides feedback
4. **Decision** - RFC is accepted, rejected, or postponed
5. **Implementation** - If accepted, track implementation progress
